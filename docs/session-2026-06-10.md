## 2. Full-game audit

A read-through of all ~7k lines (sim, network, server, client) produced a
findings report: 10 bugs, a dead-code list, optimizations, and design
observations. The bugs and dead code were then fixed (sections 3–4). Still
**open** (not requested / intentionally left):

- Server simulates cosmetic bounce particles nobody consumes
  (`MatchWorld3D.updateParticles` / `spawnBounceSparks`; `getParticles()` has
  no server-side consumer — clients spawn their own).
- Settings sliders flush `Preferences` to disk on every drag frame
  (`GameSettings.save()` called from each setter; `ConfigScreen.render` calls
  `tryDrag` per frame). Debounce on `release()`/`hide()` would fix it.
- Minor per-frame allocations: `Color.valueOf(...)` in `LoadingScreen.render`,
  `new Color(0,0,0,0)` in `ConfigScreen.TabPill.draw`, per-frame `ArrayList`
  in `MultiplayerLobbyScreen.activeButtons()`.
- `extrapolateBallY` (client dead-reckoning) bounces off the infinite table
  plane — a ball missing the table can appear to bounce mid-air for ~33 ms.
- Item effects use precedence, not stacking (WIDE_PADDLE fully cancels
  TINY_PADDLE; SLOW_MO beats FAST_SERVE).
- `ITEM_DEALT` broadcasts both hands to both clients (a modified client can
  see the opponent's items).
- Round overlay shows 2.5 s but the server parks only 2.0 s between rounds.
- The aim-ring cursor freezes while the mouse button is held (`NetInput`
  tracks `mouseMoved` but not `touchDragged`).

## 3. Bug fixes

1. **GPU leak** — libGDX never calls `Screen.dispose()`; every match leaked a
   `MatchArenaRenderer`. `NetMatchScreen.returnToMenu()` now calls
   `dispose()`; `hide()` no longer destroys renderers (pause→resume safe).
2. **Server lobby ghost** — `GameServer.MatchLobby.onLeave()` evicts dropped
   lobby connections, unlocks the mode when empty, promotes a lone second
   joiner to host. `LobbyListener` handles `onBye`/`onDisconnected`/`onError`.
   The lobby-ready snapshot in `runBestOf3` is taken under the lobby lock.
   (Repro before fix: HOST → cancel → VS BOT rejected forever / re-HOST gave
   a zombie match.)
3. **PVP serve stall** — serves now forfeit after a 20 s grace window
   (`MatchWorld3D.PVP_SERVE_TIMEOUT`); the serving player loses the point.
   Covered by `MatchWorld3DServeTimeoutTest`.
4. **Overlapping end-of-match overlays** — `onDisconnected`/`onError`/`onBye`
   no longer set `disconnected` when `matchOver` (the server closing
   connections after GAME_OVER is expected).
5. **Fly desync** — `FLY_SPAWN` / `FLY_KILLED` packets now carry the owning
   player; the client keeps `myFlies` / `oppFlies` separate, so both players
   can have flies at once and kill indices can't cross sides. Stale fly
   visuals are cleared when a new item phase starts.
6. **Pause lost the item cubes** — `ItemPhaseRenderer` lives for the screen's
   lifetime (disposed in `dispose()`, not `hide()`); also removes the OBJ
   reload hitch on every resume.
7. **VS BOT ignored `PINGPONG_SERVER_HOST`** — `Main.openMatch()` uses
   `GameConfig.resolveServerHost()` instead of a hardcoded `127.0.0.1`.
8. **Fly buzz was a no-op** — `MatchArenaRenderer.tickFlyBuzz` now rewrites
   instance transforms each frame (per-fly phase offset).
9. **Dead "UI Volume" setting removed** — slider, field, prefs key (no UI
   sounds exist).
10. **PUNCH immune to settings** — the punch blur runs even with the retro
    filter off: `RetroPostProcess.begin()` activates whenever `punchBlur > 0`,
    with stylization uniforms neutralized when the filter is disabled.

Also: the in-match status line now says "Pick your items, then click READY." /
"Waiting for opponent's items..." during the item phase instead of the
misleading "Waiting for opponent to serve...".

## 4. Dead code & robustness

- **Legacy protocol removed**: `SERVE`/`HIT` packets, `sendServe`/`sendHit`,
  `onServe`/`onHit`, `MatchWorld3D.playerHit`/`acceptClientHit`,
  `HitVelocity.sanitizeNetworkReturn` (its direction check was inverted —
  it would have rejected every legitimate hit). All returns flow through
  `CLICK` → server-side pick ray.

  hits behind the ray origin.

  `getCurrentApproachDuration`, `getBallInstance`, `DuelistState` punch-timer

  `GameSession.localName`/`buildMatchConfig`, `MatchConfig.add*` mutators,
  `GameSettings.uiVolume`. `RoomCode`'s invalid-input sentinel is now
  `LENGTH`-correct (`"???????"`).

## 5. Table model investigation (`assets/models/table/table.obj`)

### Import fixes that are IN the code

- **Per-axis fit + surface pinning** (`MatchArenaRenderer.loadTableModel`):
  the model ships with its own net, so its bbox top is the NET top, not the
  play surface. The old "bbox top → TABLE_TOP_Y" fit sank the visible
  tabletop ~1.7 world units below the gameplay plane (ball/items/flies
  floated mid-air; the procedural yellow net hovered above the model's).
  Now: length/width fit the 14×6 playfield exactly; height is scaled so the
  model's net (surface plane → bbox top) equals the gameplay `NET_HEIGHT`;
  the measured surface plane (`TABLE_SURFACE_LOCAL_Y = 0.06`) is pinned to
  `TABLE_TOP_Y`, which lands the model's net top exactly on `NET_TOP_Y`.
  The procedural net is hidden while the model's net is used
  (`tableHasOwnNet`).
- **Double-sided rendering** (`IntAttribute.createCullFace(GL_NONE)` on all
  Meshy materials — table, egg, items): the mesh has inconsistent winding
  (301 of 338 horizontal tabletop faces are CW seen from above), so back-face
  culling hid the painted top entirely.
- **Mipmaps + trilinear** on all OBJ textures (`TextureProvider`): the 2048²
  photographic atlases aliased into noise at glancing angles.

### Asset defects (measured — fix these in Blender/Meshy)

The OBJ's top is genuinely broken; model viewers hide it by being lenient:

- Local bbox: x −0.956..0.955 (length), y −0.304..0.303, z −0.633..0.631
  (width). Net: dense mesh at |x| < 0.15, y ≈ 0.10 → 0.303 (bbox top).
  Play-surface shells: y ≈ 0.058–0.065.
- **Double skin**: a plank layer and the green paint layer are welded into
  one connected component, nearly coplanar. On the x>0 half, green is
  +0.00009 above the planks (renders green); on the **x<0 half the planks
  sit 0.0011 ABOVE the paint** — any correct z-buffer renderer shows planks
  there, and the green underneath is only fragments.
- **Debris boards** lie on the surface (large low-poly faces up to y ≈ 0.13)
  and the halves are warped to slightly different heights, with a raised rim.
- Texture orientation is correct as loaded (no V-flip needed) — verified by
  pixel-sampling `table.png` at the tabletop UVs in both orientations.

Two code-side workarounds were tried and **reverted at Pedro's request**
(the model should be fixed at the asset level instead): stripping the
geometry above the surface plane, and a painted overlay slab serving as the
table top. The current build renders the model as-is (aligned, double-sided,
mipmapped) — one green half, one planked half.

**Suggested asset fix**: in Blender, delete the plank shell + debris boards,
keep one clean top sheet per half at a single height, re-export. If a future
export has no net, `loadTableModel`'s height fit (which assumes
`bbox.max.y` = net top) and `TABLE_SURFACE_LOCAL_Y` need revisiting.

## 6. Voxel table + net (replaces the Meshy table)

The broken Meshy table was replaced by a generated voxel-style asset:

- **Generator**: `tools/voxel/generate_table.py` (Python 3, stdlib only) —
  writes `assets/models/table/{table.obj, table.mtl, table.png}`. Re-run it
  after tweaking colors/geometry.
- **Authored in gameplay units** — length 14 (local X), width 6 (local Z),
  surface at y = 2.0, net top at y = 2.5, legs to the floor — so the
  loader's per-axis fit resolves to identity (`TABLE_SURFACE_LOCAL_Y` is
  now `2.0f` in `MatchArenaRenderer`).
- **Colors are exact entries of the game's 16-color palette** (Palette.java)
  so the retro filter quantizes them to themselves. Abandoned/rusty styling
  at 0.125-unit voxels: clustered rust blooms (12 damage epicenters + edge
  wear), holes punched through the tabletop with dark interiors, a chipped
  chalk border and faded centre line, rust creeping up the legs from the
  floor, a torn checkerboard net weave with a sagging tape band, pixelated
  rusty posts. The palette lives in a 64×64 PNG; every face's UVs point at
  one palette cell.
- **Gotcha encoded in the generator**: libGDX's `ObjLoader` samples `vt` v
  top-based (no flip) — the generator writes v in that convention, not the
  standard OBJ bottom-based one.
- Budget: ~6.3k verts / ~4.9k quads / ~29k indices — inside the loader's
  16-bit limits, enforced by assertions in the generator (same-color cell
  runs are merged to stay there).
- Tunables in the script: `SEED`, wear thresholds, rust color mix, hole
  density, net tear size, tape sag (set the sag factor to 0 for a taut
  tape that matches the gameplay net height exactly).

### Bunker arena + diegetic lives (Buckshot-Roulette-style scenario)

`tools/voxel/generate_arena.py` builds the room around the table —
authored in world units (floor y=0, table at origin), split into multiple
meshes to respect 16-bit index limits:

- `models/arena/arena.obj` — room shell at 1.0-unit voxels: stained
  concrete floor with baked AO skirt, paneled walls with grooves and rust
  leaks, dark ceiling, all colors hash-dithered from the game palette.
- `models/arena/props.obj` — detail pass at 0.5/0.25 voxels: ceiling
  beams, wall pipes with brackets, a blast door behind the opponent,
  hanging cone lamp over the table, dangling wires, corner crates and a
  rusty barrel, two hanging bulb racks (chains + bar + hooks) over the
  table halves, an ARSENAL on the back wall (shotgun, machete, revolver,
  knife, fire axe — pixel-art reliefs on dark-wood boards, lit by a warm
  display PointLight), and hanging chains with meat hooks on the right
  wall (vertical silhouettes that read fine at glancing angles).
- `models/arena/sign.obj` — the "PONG" neon letters as a separate mesh so
  the renderer can drive them with an animated emissive (flicker). The
  sign is mounted on the LEFT wall reading into the depth — side walls
  are nearly edge-on from the fixed camera, so flat wall art belongs on
  the back wall; the sign works on the side because it reads as glow.
- `models/arena/bulb.obj` — one life bulb, instanced 10× by code.

`MatchArenaRenderer`:
- loads the room with a plain-floor fallback; bunker lighting: low ambient,
  warm hanging-lamp PointLight over the table, red neon PointLight on the
  left wall with a slow flicker + dropout (sign emissive synced to it), a
  warm display light over the arsenal, distance fog to black (camera far
  100 → 38; fog is (dist/far)², back wall ~28 away ⇒ ~55% fogged — lower
  far for thicker fog, but keep it above ~30 or the room corners clip).
- **diegetic lives**: `setLivesDisplay(p1, p2)` lights N of 5 bulbs on the
  rack over each player's half (warm emissive when lit, dead gray when
  lost). `NetMatchScreen` feeds it the live `p1lives`/`p2lives` each frame.
  P1's rack hangs on the RIGHT of their half, P2's on the LEFT — they never
  overlap on screen, and each client sees their own lives on the right
  (the P2 camera mirrors the world). Bulb rack coordinates are shared
  constants between the renderer (`P1_BULB_X0`/`P2_BULB_X0`/`BULB_DX/Y/Z`)
  and the generator — keep them in sync.

Iteration notes (fog + readability under the retro filter):
- Ambient/directional light are distance-independent — they wash the fog
  gradient flat and the palette quantization then erases it. The fix was
  ambient 0.19 / directional 0.22 with the POINT lights carrying the scene
  (their falloff + (dist/far)² fog with far=34 is what creates depth).
- Fog mixed toward BLACK just reads as "dark room". Visible fog needs a
  visible fog color: warm dust-gray (0.105, 0.088, 0.078) — distant
  surfaces wash toward haze — plus a fake-volumetric light shaft under the
  lamp (`buildLightCone()`: additive-blended emissive cone, no depth
  write). Together they read unambiguously as dusty air.
- `models/arena/clutter.obj` (4th arena mesh): floor junk along both side
  walls — pallet stack and a tipped drum on the left, standing drum pair,
  crate stack and another tipped drum on the right, rubble mounds both
  sides. Generator helpers: `pallet()`, `drum_standing()`, `drum_tipped()`,
  `rubble(salt)`.
- Wall decor placement must respect the fixed camera's occlusion bands:
  an object at (x, z) covers the back wall around
  x · (camZ + RZ)/(camZ − z) — the bulb racks shadow back-wall bands
  x ≈ −4.9..−1.6 (P2 rack) and x ≈ 2.8..8.3 (P1 rack, mostly the door).
  Arsenal layout: axe −10.6, shotgun −9.4..−5.4, machete −0.2..3.0,
  revolver + knife stacked right of the door (8.7+).

## 7. Synthesized audio (tools/audio/generate_audio.py)

All audio is generated procedurally (pure Python stdlib, deterministic
seed). The old placeholder track (`Temporary Music from AudioMass.mp3`)
was deleted; ball/table hit mp3s remain.

- `ui_click.wav` / `ui_hover.wav` — dry blip + soft tick. Wired into every
  menu button (`MenuBaseScreen` plays click on `tryClick` and hover on the
  hover rising edge), all `ConfigScreen` widgets, and the item-phase cubes
  + READY (`NetMatchScreen` / `ItemPhaseRenderer.updateHover` now returns
  the rising edge).
- `item_use.wav` — thump + metallic shimmer, played on the server's
  ITEM_USED confirmation.
- `life_lost.wav` — bulb pop + dark falling tone + sub thud; louder when
  it's your life, quieter for the opponent's (same place as the camera
  shake in `onState`).
- `fly_buzz.wav` — seamless 1.6 s wing loop (integer-cycle components);
  `NetMatchScreen.updateFlyBuzz()` loops it while any fly is alive and
  stops it on phase end / pause (`hide()`).
- `Music/ambient_bunker.wav` — 64 s seamless ambient loop, moody/creepy:
  detuned A1 drone pair (slow beat), dark pad partials incl. a D# tritone,
  brown-noise rumble bed (circular crossfade at the seam), five sparse
  dissonant pings with echo taps, two distant metal groans. Every periodic
  component spans an integer number of cycles over the loop so it loops
  perfectly. UI sounds use the SFX volume channel (master × sfx).
